# 多态

- 静态多态
- 动态多态

## 静态多态
就是函数重载，同一个函数名，因为形参的数量、类型不同，在编译时
系统能有效识别出不同的声明和定义，进而绑定不同的函数对象，在编译
期决定。本质是name mangling，名称修饰
**名称修饰：** 例如函数void fun()和void fun(int a),函数名都是
fun，但一个带参数另一个不带参数，最终它们在编译器内部赋予了不同
的名字，比如fun和funi_1，i表示int的参数。

## 多态多态
也就是我们通常说的面向对象编程的三个要素之一，其他两个分别是**封装**、***继承**。
它的实现方法与静态多态则完成不同。

**使用场景**： 与继承结合，实现框架开发。  
**机制**：虚函数、虚函数表以及虚函数表指针。

**使用方法**：定义基类，设计好虚函数接口。子类继承基类，并实现了自己的虚函数接口
那么，在通过基类指针调用该虚函数接口时，根据传入的子类对象的不同，
调用所属对象类的实现，进而实现了“多态”的特性。

**底层原理**：编译器为每个包含虚函数的类都构建一个虚函数表，
它的每个入口是一个指向虚函数实现的指针，并且产生的对象也都隐含
一个指向虚函数表的指针成员变量。子类的虚函数表指针和基类的虚函数表指针
分别指向各自的虚函数表。在函数调用时，运行时根据这个指针指向的表查找不同
的函数地址，实现**动态绑定**。
